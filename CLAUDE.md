## Do Not Create Junk Files

## externals/ Rules

- `externals/` is a directory for user-placed reference materials. Claude must not modify it directly.
- When referencing or processing files, copy them to `code/_prototype/` first.
- Prototypes and intermediate files generated by Claude should be saved in `code/_prototype/`.

## Build & Test

### Build

```bash
cd code/tauri && cargo build
```

### Unit Tests (Rust)

```bash
cd code/tauri && cargo test --lib
```

Timer logic tests are defined inline in `code/tauri/src/timer.rs` (69 tests).

### Integration Tests

```bash
cd code/tauri && cargo test --test app_lifecycle -- --test-threads=1
```

Pattern: process startup followed by stderr log inspection. Branches based on whether Vite dev server is running.
`code/tauri/tests/app_lifecycle.rs` is a symlink to `tests/integration/app_lifecycle.rs`.

### Frontend Type Checking

```bash
cd code && npx svelte-check
```

## Testing Strategy

- Unit tests: `#[cfg(test)]` inline (timer.rs)
- Integration tests: `tests/integration/app_lifecycle.rs` (process startup + stderr log assertions)
- Frontend: type checking via `svelte-check`

## macOS-Specific Implementation

- `ActivationPolicy::Accessory` — hides from Dock
- `NSApplicationPresentationOptions` — UI lock during breaks (HideDock, HideMenuBar, DisableProcessSwitching, DisableHideApplication, DisableForceQuit)
- `activateIgnoringOtherApps` — focus management for tray popup
- `NSWindow::setLevel(1000)` + `NSWindowCollectionBehavior` — fullscreen overlay display

## Timer Architecture

- Three stages: `advance()` -> `emit timer-tick` -> `try_transition()`
- Phase completes at exactly `duration` ticks using `<` comparison
- `apply_settings()` immediately updates the duration of the current phase as well
