## チェックリスト

### クラスC（整合性問題 — 静的分析）
- [ ] C1: テスト内にソースロジックの再実装がない
  - 除外: モック/スタブ定義、テストデータ/フィクスチャ、テストヘルパー
  - 該当: ソースと同じ計算ロジック、内部アルゴリズムの複製、期待値の動的生成
- [ ] C2: テストが期待するIFとソースの公開IFが一致している

### クラスA（テスト問題 — 実行診断）
- [ ] A1: requireパスが正しい
- [ ] A2: テスト構造に問題なし（async/await、重複定義、テストデータ形式）
- [ ] A3: 環境に問題なし（Node.jsバージョン、依存パッケージ）
- [ ] A4: beforeEach/afterEachのセットアップに問題なし
- [ ] A5: テスト間の順序依存がない
- [ ] A6: モック/テストデータが正しい（ULID形式、API形式等）

### クラスB（コード問題 — 実行診断）
- [ ] B1: コードがrequire可能（CommonJS形式）
- [ ] B2: 型定義がある（必要な場合）
- [ ] B3: グローバル変数経由の暗黙依存がない
- [ ] B4: 有効な入力に対して期待値と実際値が一致

## 手順

1. テストファイルをReadで読み込む
2. SourceファイルをReadで読み込む
3. **静的分析**: クラスCチェック（C1: ロジック再実装、C2: IF不整合）
4. **実行診断**: テストを実行し、クラスA/Bチェック
5. チェックリストをfillする
6. 問題のある項目についてWhat/Why/Howを記入する
